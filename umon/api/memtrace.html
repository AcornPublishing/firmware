API FUNCTION:

mon_memtrace()

SYNOPSIS:

Similar to mon_printf() but in coordination with the monitor's "mtrace" command, the formatted string is placed into a circular buffer in RAM.

PROTOTYPE:

void   mon_memtrace(char *format,arg1, arg2, ...);

DESCRIPTION:

This function allows the application to redirect "printf-like" formatted strings to a RAM area that can be dumped later (see mtrace command).  The purpose is to allow the application to use formatted output, but not be concerned with the status of the console device (UART).  The strings are formatted and placed in a memory buffer that was previously established by the "mtrace cfg" command.  The formatting supported is the same as the mon_sprintf() function, so refer to that for limitations.

Parameters:

  • char    *format

    • pointer to a format buffer
  • argN

    • arguments referred to (if any) by format buffer).
Return:
    Void

NOTES:

  • The user should omit newlines from the formatted string passed to mon_memtrace().  To support a clean "dump" of the trace buffer, the mon_memtrace() function will automatically prepend each string with a new line and sequence number.
  • The user should not build a string that, when formatted, exceeds 128 characters on any one call to mon_memtrace().
