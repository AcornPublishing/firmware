NAME:

comnet

SYNOPSIS:

A "Telnet-to-PC-COM-Port" connection sever.

USAGE:

comnet [-b:cehp:V] {comport #}

DESCRIPTION:

This tool runs as a  server on a PC.  It looks like a telnet server (default TCP port 8200) on the PC's network side, but instead of the telnet client being connected to some shell on the PC, this server provides a bi-directional connection between the telnet client and the PC's COM port.  Although other uses can be imagined, the purpose of this tool is to allow a user to connect to a target's COM port from a remote location.
This is a "console" application (not a windows GUI), so it is run from a console window of a PC.  Options are available to the ability to connect both the local console AND the remote telnet client to the same target at the same time, and each interface will see the actions of the other.  Obviously this requires some coordination between the multiple users.

OPTIONS:

  • -b {##}

    • override default baudrate of 19200
  • -c

    • connect local PC to COM port also
  • -e

    • run with console in local-echo mode (read-only console attachment)
  • -h

    • dump verbose 'help' output
  • -p {##}

    • override default TCP port of 8200
  • -V

    • display the version (build date) of the tool

EXAMPLES:

  • comnet -b 57600  1

    • Connect to COM1 at 57600 baud.  There is no console interaction with this session.
  • comnet -e  2

    • Connect to COM2 at 19200 baud.  All character transfer is echoed to the PC console, but the user at the console cannot intervene.
  • comnet -c  1

    • Connect to COM1 at19200 baud.  The user at the console also has full read/write access to the COM port.
