NAME:

pr

SYNOPSIS:

Put a value into a register.

USAGE:

pr {regname} {value}

DESCRIPTION:

Some ports of the monitor support debugging of an application.  When the application is started (typically via the go command) it is started by loading each of the CPU registers with the values stored in a local copy that is maintained by the monitor and modifiable by the monitor user.  Upon hitting a breakpoint, the exception handler will reload the monitorÆs copy of these registers from the context of the CPU at the time of the breakpoint.  Then while the monitor is active the user has the ability to display and/or modify the value of any of these registers.  It is important to note that this command does NOT modify the current value of the register; rather, the value that is loaded as the CPU context when the application takes control.
